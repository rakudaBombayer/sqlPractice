検索結果を並び替える(ORDER BY)

SELECT shohin_id, shohin_mei, hanbai_tanka, shiire_tanka FROM Shohin

ORDER BY hanbai_tanka; ASC(明示しないとデフォルトで昇順)


販売単価の低い順(昇順)に並べる


SELECT shohin_id, shohin_mei, hanbai_tanka, shiire_tanka FROM Shohin

ORDER BY hanbai_tanka DESC;(降順)

OREDER BYは最後に処理される


SELECT shohin_id, shohin_mei, hanbai_tanka, shiire_tanka FROM Shohin

ORDER BY hanbai_tanka, shohin_id;

NULLは先頭、末尾か、どっちかわからないがまとめて出る

ORDER BY hanbai_tanka DESC, shohin_id DESC;
昇順、降順はそれぞれにつける


SELECT shohin_id AS id, shohin_mei, hanbai_tanka AS ht, shiire_tanka FROM Shohin
ORDER BY ht, id;

FROM →WHERE →GROUP →BY →HAVING →SELECT →ORDER BY(処理の順番)はあくまでイメージ
この例は特例→SELECT →ORDER BYはイメージを持たないほうがいい



AS 別名を使えるのはORDER BYのみ

SELECT shohin_mei, hanbai_tanka, shiire_tanka FROM Shohin

ORDER BY shohin_id;


SELECT shohin_bunrui, COUNT(*) FROM Shohin GROUP BY shohin_bunrui 

ORDER BY COUNT(*);(COUNTごとの昇順に並び替えている)


練習問題3.2

SELECT shohin_bunrui, SUM(hanbai_tanka), SUM(shiire_tanka)
FROM Shohin
GROUP BY shohin_bunrui
HAVING SUM(hanbai_tanka) >= SUM(shiire_tanka)*1.5;
